# AI WorkBench 可点击元素覆盖矩阵

每个可见按钮、菜单、Tab、表单、弹窗、下载、删除、刷新、确认动作都必须覆盖正常、异常、边界三类用例。展示但不可点、禁用无原因、点击无反馈、结果不可读均为缺陷。

| 页面 | 控件/动作 | 正常用例 | 异常用例 | 边界用例 | 确认/审计 |
| --- | --- | --- | --- | --- | --- |
| Workbench | 发送消息 | 输入主机压力问题并发送 | 空输入、AI 不可用、网络失败 | 超长、emoji、XSS、SQL、命令注入 | 危险命令必须确认并回传 |
| Workbench | 模型选择 | 切换可用模型 | 模型为空、Provider 失效 | 刷新后保持选择 | 不展示明文 Key |

| Catpaw | 新增/编辑/删除凭证 | SSH/WinRM 凭证保存 | 空密码、错端口、错协议 | 重复名称、超长备注 | 凭证脱敏，删除确认 |

| Catpaw | 探针对话 | 连接、发送、断开 | 未选探针、连接失败 | 超长输出、危险命令 | 确认状态回传聊天 |
| Settings AI | 新增/保存 Provider | 正确 Key/Base URL | 空 Key、占位符、401/429 | 多模型、默认切换 | Key 脱敏 |
| Settings Datasource | 保存/健康检查 | Prometheus/MySQL/Redis 可用 | URL 错、认证错、超时 | target 离线但历史数据存在 | 错误原因可读 |

## MCP 验收规则

- 每个控件必须能被 Playwright MCP 定位。
- 禁用控件必须有 tooltip、说明文字或相邻提示解释原因。
- 点击后必须出现 loading、toast、弹窗、页面状态变化或错误提示。
- 删除、卸载、清理、远程执行必须验证取消和确认两条路径。
- 页面不得出现 `乱码替换符`、`???`、未解释问号、明文 API Key、明文密码。
