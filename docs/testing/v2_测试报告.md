## 七、已知问题

1. **语义搜索 score 显示为 0.000（中等）**
   - 现象：搜索 API 返回的 `score` 字段在多数命中条目上显示为 `0.000`。
   - 影响：仅影响展示，命中顺序与召回正确。
   - 原因：当前 `engine=fulltext` 时未填充归一化 score。
   - 处理建议：在 fulltext 引擎层增补 `relevance / max_relevance` 归一化输出。

2. **当前 engine=fulltext 而非 hybrid（中等）**
   - 现象：尽管配置写入 `engine=hybrid`，实际查询日志显示走的是 fulltext 路径。
   - 原因：DashScope embedding api_key 在自动化环境是 placeholder，hybrid 在初始化阶段 fallback 到 fulltext。
   - 处理建议：真实 api_key 注入后重新触发 reindex，验证 hybrid 与 BM25 切换链路。

3. **Reranker 网络重试日志冗余（低）**
   - 现象：API 模式下偶发 1 次 503，重试后成功，但日志里出现 stack trace 噪声。
   - 处理建议：将 retry 视作 debug 级别，仅最终失败时打印 error。
