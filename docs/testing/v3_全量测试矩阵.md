## 第二部分：工作流节点测试矩阵

> 基于 `api/internal/workflow/node/registry.go` 注册的 20 种节点类型

| `start` | 接收工作流输入参数 → 写入变量池并输出到下游 | 无输入参数时变量池为空，正常启动 | 不适用（瞬时节点） | 输入参数超过 100 个键值对；值为嵌套 JSON 对象 |
| `end` | 接收上游变量 → 组装最终输出返回调用方 | 无上游变量时输出空对象 | 不适用（瞬时节点） | 输出数据量超过 10MB；输出含二进制数据 |
