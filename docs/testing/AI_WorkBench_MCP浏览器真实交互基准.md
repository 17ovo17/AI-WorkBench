# AI WorkBench MCP 浏览器真实交互基准

MCP 浏览器测试是用户界面验收主线。API 测试只能证明后端可用，不能替代用户真实点击、输入、确认、下载、刷新和错误恢复。

## 执行原则

- 每个页面必须通过 MCP 打开真实路由并保存快照和截图。
- 每个可见按钮、Tab、下拉、表单、删除、下载、确认弹窗至少真实操作一次。
- 每个高风险动作必须分别测试取消确认和确认执行。
- 每个禁用按钮必须验证禁用原因；没有原因按 P1。
- 页面出现 `����`、`???`、明文密钥、无原因问号、点击无反馈，按 P1。

## 页面覆盖

| 页面 | 用户目标 | MCP 操作 |
| --- | --- | --- |
| `/workbench` | 完成智能问答和工具调度 | 选择模型、发送 IP 压力问题、追问、危险命令确认取消 |
| `/diagnose` | 完成诊断报告闭环 | 输入 IP、发起诊断、查看报告、下载、删除取消、删除确认 |
| `/alerts` | 完成告警处置 | 筛选、打开详情、触发诊断、标记恢复、刷新验证 |
| `/topology` | 完成业务拓扑建模 | 新增业务、输入 IP/端口、AI 发现、折叠观测层、删除节点/业务 |
| `/catpaw` | 完成探针生命周期入口验证 | 凭证管理、系统切换、生成/安装入口、模式切换、探针对话连接/发送/断开 |
| `/settings/ai` | 完成 AI 配置 | 新增 Provider、模型管理、设默认、保存、健康检查、脱敏验证 |
| `/settings/datasource` | 完成数据源配置 | 保存 Prometheus/MySQL/Redis、健康检查、错误恢复 |

## 证据文件

- `mcp-browser-summary.md`：每页测试结论和 P0/P1/P2/P3。
- `mcp-click-coverage.json`：每个控件的定位方式、点击结果、反馈状态。
- `mcp-page-snapshots/`：页面无障碍快照。
- `mcp-screenshots/`：关键状态截图。
- `mcp-console.json`：控制台错误与警告。
- `mcp-network.json`：关键接口请求与响应摘要。

## 定调规则

- API 通过但 MCP 失败：以 MCP 结果为准，不能标记通过。
- MCP 未覆盖的按钮：状态为“待测”，不能标记完成。
- MCP 发现用户无法理解的禁用/失败状态：至少 P2；核心链路为 P1。
