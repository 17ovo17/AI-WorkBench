# CLAUDE.md — AI WorkBench 开发准则

## 语言规范
- 所有对话、文档、代码注释使用简体中文
- 文档使用 Markdown 格式

## Claude + Codex 协作模式

### 总原则
- **问题驱动优先于流程驱动**：追求充分性而非完整性
- **默认自动执行，不询问用户**：仅 5 类例外需确认（删核心配置/数据库破坏/Git push/连续 3 次同错/用户要求）
- **渐进式迭代**：保持每次改动可编译、可验证

### 角色定义

**Claude Code（主AI）— 规划者 + 执行者**
- 需求理解和疑问识别
- 任务规划和拆解
- 最终决策确认（基于 Codex 建议）
- 决策记录留痕（operations-log.md）
- 简单修改直接执行（≤10 行核心逻辑）

**Codex（支持AI）— 分析者 + 审查者**
- 深度推理分析（核心优势）
- 全面代码检索和上下文收集
- 编码实现（>10 行核心逻辑）
- 复杂逻辑设计（算法、架构决策）
- 质量审查评分（代码审查、风险识别）
- 技术建议输出（非决策，是输入）

### 强制规则
1. **编码前分析**：编码前必须调用 Codex 获取分析报告
2. **复杂度委托**：>10 行核心逻辑必须由 Codex 编码
3. **质量审查**：所有代码变更必须经 Codex 审查
4. **决策留痕**：Claude 推翻 Codex 建议时必须记录充分理由

### Codex 调用方式
- **首选**：MCP 工具（mcp__codex__codex）
- **降级**：Bash CLI（`codex --prompt="..." --approval-mode full-auto`）
- 会话管理：通过 task_marker `[TASK_MARKER: YYYYMMDD-HHMMSS-XXXX]` 管理

### 审查协作规范
- Codex 输出审查报告：技术维度 + 战略维度 + 综合评分(0-100) + 明确建议
- Claude 决策规则：≥90分且"通过"→直接确认；<80分且"退回"→直接退回；80-89分→仔细审阅

### 工作文件路径
所有工作文件写入 `<project>/.claude/`（禁止写入 `~/.claude/`）

## 架构优先级
- "标准化 + 生态复用"拥有最高优先级
- 禁止新增自研方案，除非已有实践无法满足需求

## 代码质量标准
- 动态语言每文件 ≤ 300 行，静态语言每文件 ≤ 400 行
- 单函数不超过 50 行
- 禁止复制粘贴、God Function、Magic Number、硬编码 URL/IP/密码
- 禁止跨层调用、循环导入

## 强制验证机制
- 拒绝一切 CI、远程流水线验证，所有验证由本地 AI 自动执行
- 每次改动必须提供可重复的本地验证步骤
- 连续三次验证失败必须暂停实现，回到需求和设计阶段复盘

## Run & Debug
- 所有启停操作使用 scripts/ 下的 .sh 脚本
- 代码修改后必须同步到 WSL /opt/ai-workbench，Go 在 WSL 内编译
- 日志统一输出到 logs/ 目录

## 项目技术栈
- 后端：Go 1.21 + Gin
- 前端：Vue 3.5 + Vite 5 + Element Plus + Tailwind CSS v4
- 数据库：MySQL 8 + Redis
- 监控：Prometheus
- 工作流引擎：Go 原生 DAG（20 种节点类型）
- 搜索引擎：BM25 + 向量 + RRF + Reranker
